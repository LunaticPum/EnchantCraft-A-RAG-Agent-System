# 检索页完善 + Kafka 异步 Embedding 联调

## 检索页对接真实 API

V5.4 将检索实验室从 Mock 数据切换为真实后端 API：

```typescript
// SearchPage.tsx
const handleSearch = async () => {
  setSearching(true);
  const data = await api.search(query, 5, strategy, true);  // 调后端 RAG 检索
  setResults(data);
};
```

三种策略均可切换——`SEMANTIC`（向量相似度）、`KEYWORD`（中文分词全文检索）、`HYBRID`（RRF 融合）。每条结果支持点击展开查看完整 Markdown 渲染内容。

### LaTeX 公式支持

LLM 输出的 Markdown 中可能包含 LaTeX 公式（如 `$$\mathrm{Slot} = \mathrm{CRC16}(\mathrm{Key})\bmod 16384$$`）。前端通过 `remark-math` + `rehype-katex` 渲染：

```tsx
<ReactMarkdown remarkPlugins={[remarkGfm, remarkBreaks, remarkMath]}
               rehypePlugins={[rehypeKatex]}>{md}</ReactMarkdown>
```

## Kafka 异步 Embedding 联调

V5.3 重构完 Agent 后，Kafka 异步索引链路也正式联调通过。

### Kafka 部署踩坑

| 镜像 | 结果 | 原因 |
|------|------|------|
| `apache/kafka:3.7.1` | ❌ 写权限错误 | 容器内 `appuser(uid=1000)` 无法写 `/var/lib/kafka/data` |
| `apache/kafka:3.7.1` + `user: root` | ✅ 成功 | 以 root 用户运行，写权限问题解决 |
| `bitnami/kafka:4.0` | ✅ 可运行 | 正常，但 topic 自动创建有时不生效 |

最终配置：`apache/kafka:3.7.1` + `user: root` + `KAFKA_AUTO_CREATE_TOPICS_ENABLE=true`。

### 自动建 topic

Kafka broker 开启 `auto.create.topics.enable=true` 后，producer 第一次发送消息时会自动创建 topic。不需要手动执行 `kafka-topics.sh --create`。

Producer 端加 `max.block.ms=3000` 防止 topic 不存在时阻塞过久：

```yaml
spring:
  kafka:
    producer:
      properties:
        max.block.ms: 3000   # 默认 60000，降低超时等待
```

### 全链路验证

```
POST /upload → DocumentServiceImpl.uploadDocument()
  ├─ ① MySQL 落库（事务内）
  ├─ ② KafkaIndexingProducer.sendChunkEmbedMessage()
  │      → broker 自动创建 topic document.indexing
  │      → message_job 记录 PENDING
  │
  Kafka Consumer 异步消费
  ├─ ③ IndexingConsumer.onMessage()
  │      → documentService.embedDocumentChunks()
  │      → PgVectorStore.addAll()
  │      → message_job → COMPLETED
  │
  MessageRetryJob 兜底
  └─ 每 30s 扫描 PENDING/FAILED → 重试
```

## 前端状态持久化

Agent 对话状态通过 `AgentStateContext` 提升到应用级别，路由切换不丢失：

```tsx
// App.tsx
<AgentStateProvider>
  <BrowserRouter>...</BrowserRouter>
</AgentStateProvider>

// AgentPage.tsx - 从 Context 获取状态
const { messages, setMessages, sending, sessionId } = useAgentStore();
```
